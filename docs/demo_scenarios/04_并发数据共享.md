# 场景 4：并发数据共享

## 场景目标

演示多窗口模式下 MASTER 与 CLIENT 通过命名管道共享同一内核状态。窗口 A 创建数据后，窗口 B 可以查看同一状态并继续提交命令；窗口 A 再次查看时能看到窗口 B 的修改。

## 核心操作

1. 打开窗口 A，作为第一个实例启动，通常为 MASTER。
2. 在窗口 A 注册并登录 `share` 用户，创建初始进程。
3. 打开窗口 B，作为后续实例启动，通常为 CLIENT。
4. 在窗口 B 查看当前用户和系统总览。
5. 在窗口 B 创建子进程。
6. 回到窗口 A 再次查看系统总览。

## 菜单操作路径

菜单名称以当前 `MenuConsole` 为准：

1. 窗口 A：用户管理 → 注册用户
2. 窗口 A：用户管理 → 登录用户
3. 窗口 A：进程管理 → 创建进程
4. 窗口 A：系统总览 → 查看系统全局总览
5. 窗口 B：用户管理 → 查看当前用户
6. 窗口 B：系统总览 → 查看系统全局总览
7. 窗口 B：进程管理 → 创建进程
8. 窗口 A：系统总览 → 查看系统全局总览

## 原始命令脚本

窗口 A：

```text
register share 123456
login share 123456
create_pcb init 64 0 20
overview
```

窗口 B：

```text
whoami
overview
create_pcb client_worker 128 5 12 1
```

窗口 A：

```text
overview
```

## 预期输出

- 窗口 A 能创建 `init`。
- 窗口 B 的命令通过 NamedPipeClient 发送到 MASTER。
- 窗口 B 创建 `client_worker` 后，窗口 A 的 `overview` 能看到该进程。
- CLIENT 退出只关闭当前窗口，不关闭 MASTER 内核。

## 验收标准

- 窗口 A 和窗口 B 看到的是同一份内核状态。
- 窗口 B 不直接创建 Kernel，不直接读写快照文件。
- 窗口 A 再次查看总览时包含窗口 B 创建的真实进程。
- MASTER 仍可继续处理命令。

## 截图建议

- 截取窗口 A 初始 `overview`。
- 截取窗口 B 创建 `client_worker` 的结果。
- 截取窗口 A 最终 `overview`，突出新增进程。

## 常见问题排查

- 如果窗口 B 无法连接，确认窗口 A 的 MASTER 仍在运行。
- 如果窗口 B `whoami` 显示未登录，可在窗口 B 执行 `login share 123456` 后继续。
- 如果创建子进程失败，确认父进程 PID `1` 仍存在。
