# 场景 1：调度动态性

## 场景目标

演示多级反馈队列在自动调度启动与停止期间的动态变化。重点观察进程创建后进入不同就绪队列，自动调度推进 CPU 时间后，进程状态、队列位置和系统总览中的 MLFQ 视图如何同步变化。

## 核心操作

1. 注册并登录演示用户 `demo`。
2. 创建三个不同优先级的 CPU 型进程。
3. 查看初始就绪队列。
4. 启动自动调度，让调度器运行一小段时间。
5. 停止自动调度，查看进程列表和系统总览。

## 菜单操作路径

菜单名称以当前 `MenuConsole` 为准：

1. 用户管理 → 注册用户
2. 用户管理 → 登录用户
3. 进程管理 → 创建进程
4. 进程管理 → 查看就绪队列
5. 调度管理 → 启动自动调度
6. 调度管理 → 停止自动调度
7. 进程管理 → 查看进程列表
8. 系统总览 → 查看系统全局总览

## 原始命令脚本

```text
register demo 123456
login demo 123456
create_pcb cpu_a 64 0 20
create_pcb cpu_b 64 5 18
create_pcb cpu_c 64 10 16
readyq
start_sched
stop_sched
list_pcb
overview
```

## 预期输出

- `register` 和 `login` 返回成功信息。
- `create_pcb` 输出新建进程的 PID。
- `readyq` 中能看到 `cpu_a`、`cpu_b`、`cpu_c` 分布在对应队列。
- `start_sched` 输出调度器已启动。
- `stop_sched` 输出调度器已停止。
- `list_pcb` 中 CPU 时间或状态可能已经变化。
- `overview` 包含 `System Overview`、`System Summary`、`Process Table`、`Process Tree`、`Memory Map`、`MLFQ`。

## 验收标准

- 自动调度启动和停止命令均成功。
- 停止后系统仍可继续响应命令。
- 进程列表与系统总览展示的进程数据一致。
- MLFQ 队列展示来自真实进程快照，不出现固定假数据。

## 截图建议

- 截取 `readyq` 初始队列输出。
- 截取 `start_sched` 与 `stop_sched` 的命令结果。
- 截取 `overview` 中的进程表和 MLFQ 区域。

## 常见问题排查

- 如果提示需要登录，先执行 `login demo 123456`。
- 如果自动调度变化不明显，在 `start_sched` 后等待 1 到 2 秒再执行 `stop_sched`。
- 如果进程已经完成，可重新创建进程或换一个新用户重跑场景。
