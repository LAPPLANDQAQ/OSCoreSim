# 场景 6：终端可视化

## 场景目标

演示 `overview` 在终端中集中展示系统摘要、程序表、进程树、内存分区图、MLFQ 队列和说明信息。该场景适合作为课程答辩中的总览展示入口。

## 核心操作

1. 注册并登录 `view` 用户。
2. 创建一个父进程和两个子进程。
3. 手动分配一块内存。
4. 写入一个虚拟文件。
5. 执行一次单步调度。
6. 查看系统总览。

## 菜单操作路径

菜单名称以当前 `MenuConsole` 为准：

1. 用户管理 → 注册用户
2. 用户管理 → 登录用户
3. 进程管理 → 创建进程
4. 内存管理 → 手动分配内存
5. 虚拟文件系统 → 写入文件
6. 调度管理 → 单步调度
7. 系统总览 → 查看系统全局总览

## 原始命令脚本

```text
register view 123456
login view 123456
create_pcb init 64 0 20
create_pcb shell 128 5 12 1
create_pcb logger 32 10 8 1
alloc 100
write_file view.txt visualization test
step
overview
```

## 预期输出

- `overview` 顶部显示 `系统总览 / System Overview`。
- 系统摘要中显示当前用户、调度器状态、内存使用、进程数量和 VFS 文件数量。
- 程序表中显示 `init`、`shell`、`logger`。
- 进程树中显示父子关系。
- 内存分区图中包含进程内存和手动分配内存。
- MLFQ 区域显示当前队列快照。

## 验收标准

- 六个总览章节完整出现。
- 输出包含 `System Overview`、`System Summary`、`Process Tree`、`Memory Map`、`MLFQ` 等英文关键词。
- 展示的数据与当前用户真实创建的进程、内存和文件数量一致。
- `overview` 只读，不改变进程、内存、调度或文件状态。

## 截图建议

- 截取 `overview` 的系统摘要区域。
- 截取程序表和进程树区域。
- 截取内存分区图和 MLFQ 区域。
- 如果终端窗口较小，建议分段截图。

## 常见问题排查

- 如果 `write_file` 失败，确认已经登录 `view` 用户。
- 如果 `create_pcb shell ... 1` 失败，确认 `init` 的 PID 是 1；若不是，使用实际父进程 PID。
- 如果总览内容过长，可扩大终端窗口或向上滚动查看完整输出。
