# 场景 5：内存碎片治理

## 场景目标

演示手动内存分配、释放、碎片统计、内存紧缩以及分配算法切换。重点观察释放中间块后产生外部碎片，执行紧缩后内存分区如何变化。

## 核心操作

1. 注册并登录 `mem` 用户。
2. 连续分配 100KB、120KB、80KB 手动内存。
3. 查看内存分区。
4. 释放起始地址为 100KB 的中间块。
5. 查看碎片统计。
6. 执行内存紧缩，再次查看分区和统计。
7. 切换 BEST_FIT 与 WORST_FIT，继续分配内存。

## 菜单操作路径

菜单名称以当前 `MenuConsole` 为准：

1. 用户管理 → 注册用户
2. 用户管理 → 登录用户
3. 内存管理 → 手动分配内存
4. 内存管理 → 手动分配内存
5. 内存管理 → 手动分配内存
6. 内存管理 → 查看内存分区
7. 内存管理 → 释放手动内存
8. 内存管理 → 查看内存统计
9. 内存管理 → 内存紧缩
10. 内存管理 → 切换分配算法

## 原始命令脚本

```text
register mem 123456
login mem 123456
alloc 100
alloc 120
alloc 80
show_mem
free_mem 100
show_mem
mem_stat
compact
show_mem
mem_stat
set_alloc_algo BF
alloc 50
set_alloc_algo WF
alloc 30
show_mem
```

## 预期输出

- 三次 `alloc` 输出不同起始地址。
- `free_mem 100` 释放中间的 120KB 块。
- `mem_stat` 显示总内存、已用内存、空闲内存和外部碎片率。
- `compact` 输出紧缩结果，后续 `show_mem` 中空闲块更集中。
- `set_alloc_algo BF` 和 `set_alloc_algo WF` 成功切换算法。

## 验收标准

- 手动内存块的释放只影响当前用户可操作的块。
- 紧缩前后 `show_mem` 的分区起始地址或空闲块布局发生合理变化。
- 分配算法切换后仍能继续分配内存。
- 所有内存输出来自真实 MemoryManager 状态。

## 截图建议

- 截取释放前的 `show_mem`。
- 截取释放后的 `show_mem` 和 `mem_stat`。
- 截取 `compact` 后的 `show_mem`。
- 截取算法切换后的最终 `show_mem`。

## 常见问题排查

- 如果 `free_mem 100` 失败，检查第二次 `alloc 120` 是否确实从 100KB 开始。
- 如果内存布局受历史演示影响，可换新用户或重启后重跑。
- 如果紧缩变化不明显，增加更多分配和释放步骤制造碎片。
